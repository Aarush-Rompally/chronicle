# Chronicle: a text-based EHR system with Admin, Physician, and Patient roles

Judy Faulkner, CEO and founder of Epic Systems, discussed a problem she was once tasked with that grew into Chronicles, the very core of what is now a sprawling giant in the field of electronic healthcare. Inspired by the idea of tracking health electronically across past, present, and future, I put together and executed an object-oriented design for an EHR system, serving several roles within the healthcare ecosystem that interact dynamically through this database, named 'Chronicle' after Faulkner's project. This project was techincally complex and raised important considerations, like patient autonomy with regard to health history, and privacy. I hope that you take a second to dig into the code or run it in your terminal and interact with the interface.

NOTE: when beginning the program, no physicians or patients are defined. Admin ID is *. From there, instances of other roles can be added to the system.
